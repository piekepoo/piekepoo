Video Killed the ASCII Radio



w/ Joëlle Zweekhorst and Tinka Zorge

Summary

As an homage to old media, different old media are integrated into one machine: ASCII Radio. This radio plays messages that visitors can send in through a web server. These messages will be translated as spoken ASCII code. Someone on the receiving end of the radio can translate the spoken numbers back to the original message using only a typewriter and a an ASCII table with translations. Developed with Python.

Concept

Imagine: it’s 2018 and you’re gossiping about secrets from people you know through a social networking site. Now, imagine that it’s not 2018, but maybe somewhere in the 1960s. You don’t have a social networking site. You don’t even have a computer. You may use computer language, but not a computer. So how would that go? We came up with the idea of an ASCII Radio, a device that translates secret messages/gossip into voice recordings. By translating the spoken ASCII numbers back to the predefined letters, the messages can be decoded and the gossip will become visible. Basically, you’re taking over the computers job, because while usually translations happen within the computer, you’re the one doing the translation this time, but with a different medium.

Motivation

We really liked the idea of secrets, both as in social secrets but also the fact that the computer uses a secret language that humans usually don’t understand. We came up with the idea to combine these together, so by having social secrets being translated through a secret language. Our idea was the omit the computer itself, as it was most interesting to see what would happen if we just used this concept and used different media to find a way to do things what computers are so typical of.

Implementation

We wrote a code in Python, that played certain sound files for each letter in an incoming message. To get sound files of numbers, we used a Google Translator and saved those files into the folder. We loaded the code into a Raspberry Pi, that was connected to a speaker of a jukebox. We created a poster of the ASCII Table that was used to translate the numbers back to letters. The letters could be typed into a typing machine to decode the incoming messages. For the Raspberry Pi to receive incoming messages, these were posted on a website that we made, and from this website the messages were stored in a SQL database on the computer. Python could read these messages from the database.
